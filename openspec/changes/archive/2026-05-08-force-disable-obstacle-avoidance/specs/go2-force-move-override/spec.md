# Go2 强制移动覆盖规格

## 新增需求

### 需求：手动强制移动覆盖手势

Command Center 必须提供一个显式的手动控制手势，并且只在操作员主动发出移动指令时请求临时强制移动覆盖。

#### 场景：操作员在移动时按住覆盖手势

- **给定** 键盘控制处于启用状态
- **当** 操作员同时按住 Ctrl+Shift 和一个移动键
- **则** 客户端必须请求启用强制移动覆盖
- **并且** 客户端必须以降低后的覆盖速度继续发送移动指令

#### 场景：操作员释放覆盖手势

- **给定** 强制移动覆盖处于启用状态
- **当** 操作员释放覆盖手势或停止移动
- **则** 客户端必须请求禁用强制移动覆盖

### 需求：typed 覆盖传输

Web 可视化模块必须将强制覆盖 Socket.IO 事件转换为 typed DimOS stream。

#### 场景：Web 客户端启用覆盖

- **给定** Web 客户端发送带有 `enabled: true` 的 `force_move_override` 事件
- **当** WebSocket 可视化模块收到该事件
- **则** 它必须在 `force_move_override` 上发布 `Bool(data=True)`

#### 场景：Web 客户端禁用覆盖

- **给定** Web 客户端发送带有 `enabled: false` 的 `force_move_override` 事件
- **当** WebSocket 可视化模块收到该事件
- **则** 它必须在 `force_move_override` 上发布 `Bool(data=False)`

### 需求：临时 Go2 避障覆盖

Go2 连接模块必须只在强制移动覆盖生效期间关闭避障，并且必须在覆盖结束时恢复配置中的避障状态。

#### 场景：覆盖被启用

- **给定** Go2 连接模块在 `force_move_override` 上收到 `Bool(data=True)`
- **当** 状态从禁用变为启用
- **则** 连接模块必须调用 `set_obstacle_avoidance(False)`

#### 场景：覆盖被禁用

- **给定** 强制移动覆盖处于启用状态
- **当** Go2 连接模块收到 `Bool(data=False)`
- **则** 连接模块必须发送零速度移动指令
- **并且** 必须将避障恢复为配置值

#### 场景：覆盖刷新超时

- **给定** 强制移动覆盖处于启用状态
- **当** 覆盖超时前没有收到刷新
- **则** Go2 连接模块必须禁用强制移动覆盖
- **并且** 必须将避障恢复为配置值

#### 场景：Go2 连接停止

- **给定** 强制移动覆盖处于启用状态
- **当** Go2 连接停止
- **则** 它必须在完成关闭前禁用强制移动覆盖
