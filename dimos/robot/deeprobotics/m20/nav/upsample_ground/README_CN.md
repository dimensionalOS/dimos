# M20 地面点云上采样技术研发方案

[English version](/dimos/robot/deeprobotics/m20/nav/upsample_ground/README.md)

## 1. 文档目的

本文档定义 DeepRobotics M20 导航栈中单层地面点云上采样模块的设计与交付方案。该模块在生成代价地图之前填补已观测地面数据中的小范围空洞，同时避免机器人重复经过同一区域时不断追加重复地面点。

初始运行假设为室内或局部平面环境，局部感兴趣区域为 `5 m x 5 m`，输出栅格间距为 `0.05 m`。多层地图明确不在本方案范围内。

## 2. 背景

当前 M20 导航流水线定义在 [`m20_simple_nav.py`](/dimos/robot/deeprobotics/m20/nav/m20_simple_nav.py) 中：

1. `slam_aligned_points` 提供世界坐标系下的当前帧点云。
2. `RayTracingVoxelMap` 累积表面体素并发布 `global_map`。
3. `CostMapper` 使用高度梯度将 `global_map` 转换为 `global_costmap`。
4. `ReplanningAStarPlanner` 基于生成的代价地图进行路径规划。

`RayTracingVoxelMap` 只发布表面点，其内部计算的射线 miss 不会在输出 `PointCloud2` 中表示为自由空间栅格。模块接口定义在 [`dimos/mapping/ray_tracing/module.py`](/dimos/mapping/ray_tracing/module.py#L28) 中，原生更新与发布逻辑位于 [`dimos/mapping/ray_tracing/rust/src/main.rs`](/dimos/mapping/ray_tracing/rust/src/main.rs#L29) 和 [`voxel_ray_tracer.rs`](/dimos/mapping/ray_tracing/rust/src/voxel_ray_tracer.rs)。

地面点可能比较稀疏，也可能被 health 门控和射线清除删除。当地面点缺失时，[`occupancy.py`](/dimos/mapping/pointclouds/occupancy.py) 中基于高度的实现无法可靠地将相应栅格标记为已知和可通行。

## 3. 目标

- 从每一帧世界坐标系点云中估计局部地面表面。
- 以 `0.05 m` 间距填补具有充分观测支撑的地面区域中的小空洞。
- 在机器人运动过程中，以世界坐标持久化经过确认的合成地面。
- 每个世界 XY 栅格单元最多保存一个合成地面点。
- 机器人重复经过同一位置时，更新或替换已有地面单元。
- 将合成地面与最新且具有权威性的 `RayTracingVoxelMap.global_map` 快照融合。
- 保证真实传感器点和射线地图点具有更高的数据权威级别。
- 防止合成地面掩盖低矮障碍物、墙体、坑洞或落差边缘。
- 保证所有生成数据均不进入 `RayTracingVoxelMap` 的反馈路径。
- 提供用于衡量拟合质量、状态增长、处理延迟和拒绝原因的诊断信息。

## 4. 方案架构

在 `RayTracingVoxelMap` 和 `CostMapper` 之间增加 `UpsampleGround` 模块。

| 端口 | 方向 | 类型 | 来源或消费者 | 用途 |
|---|---|---|---|---|
| `current_frame` | 输入 | `PointCloud2` | `slam_aligned_points` | 估计当前局部地面平面 |
| `global_map` | 输入 | `PointCloud2` | `RayTracingVoxelMap.global_map` | 具有权威性的真实累积表面快照 |
| `odometry` | 输入 | `Odometry` | `slam_odom` | 定义机器人相对 ROI 和高度范围 |
| `global_map_upsample_ground` | 输出 | `PointCloud2` | `CostMapper.global_map` | 真实全局地图与唯一合成地面的融合结果 |
| `ground_patch` | 可选输出 | `PointCloud2` | Rerun 或诊断模块 | 单独显示模块生成的合成地面 |

预期数据流如下：

1. `slam_aligned_points`、`slam_odom` 和 `RayTracingVoxelMap.global_map` 分别进入按时间戳升序排列的有界队列。
2. 模块取 `global_map` 队首作为同步基准，为其匹配最近的当前帧和里程计样本。
3. 同步成功后，当前 `global_map` 触发一次地面估计和融合输出。
4. 同步尚缺少未来样本时保留基准帧并等待；确认无法满足同步条件时不处理当前帧，但仍将当前真实 `global_map` 与已有历史合成地面融合后输出 `global_map_upsample_ground`。
5. 当前帧更新持久化合成地面状态。
6. 最新真实 global map 快照与合成地面状态融合。
7. `CostMapper` 消费 `global_map_upsample_ground`，不再直接消费射线地图输出。

融合输出绝不能被路由到 `RayTracingVoxelMap`，也不能作为历史数据重新输入 `UpsampleGround`。

## 5. 核心状态模型

`UpsampleGround` 不得累积真实 `global_map`。它本身已经是完整的射线追踪历史快照，在模块中仅用于当前一次融合。

只有合成地面需要持久化。合成地面存储在与世界坐标对齐的 XY 哈希栅格中：

| 字段 | 类型 | 含义 |
|---|---|---|
| `ix` | `int` | `floor(world_x / ground_resolution)` |
| `iy` | `int` | `floor(world_y / ground_resolution)` |
| `z` | `float` | 当前融合后的地面高度 |
| `confidence` | `float` | 已保存高度的置信度 |
| `observations` | `int` | 已接受的更新次数，具有饱和上限 |
| `last_seen_ts` | `float` | 最近一次接受观测的时间戳 |
| `fit_id` | `int` | 最近更新该单元的平面拟合标识 |

状态键为 `(ix, iy)`。由于多层运行不在本方案范围内，每个世界 XY 单元最多只有一个合成地面高度。

必须保持以下不变量：

`synthetic_ground_count == number_of_unique_world_xy_keys`

重复经过只会更新已有单元，绝不会在同一 XY 坐标追加第二个合成点。

## 6. 地面估计算法

### 6.1 输入校验与同步

- 拒绝不包含有限 XYZ 点的点云。
- 要求 `current_frame.frame_id` 和 `global_map.frame_id` 表示同一个世界坐标系。
- 分别为 `current_frame`、`global_map` 和 `odometry` 创建有界队列，所有队列按消息时间戳升序排列；乱序到达的数据必须按时间戳插入，不能按到达顺序匹配。
- 始终以 `global_map` 队首消息的时间戳 `t_ref` 作为本轮同步基准，并分别在 `current_frame` 和 `odometry` 队列中执行相同的匹配算法。
- 融合输出沿用 `global_map.ts` 和 `global_map.frame_id`。

对任一辅助队列执行以下匹配规则：

1. 队列为空时返回“等待”，不执行地面估计，也不消费当前基准 `global_map`。
2. 队首时间戳等于 `t_ref` 时，直接选择队首数据。
3. 队首时间戳小于 `t_ref` 时，从队首向后遍历，直到找到第一个时间戳大于 `t_ref` 的样本。
4. 如果遍历到队尾仍没有大于 `t_ref` 的样本，说明后续数据可能尚未到达，返回“等待”；保留当前基准 `global_map`，待新数据进入队列后重新同步。
5. 如果找到了第一个大于 `t_ref` 的样本，比较它与前一个小于 `t_ref` 的样本到 `t_ref` 的绝对时间差，选择时间差更小者；时间差相同时固定选择较早样本，保证结果确定。
6. 队首时间戳大于 `t_ref` 时，仅当 `queue_head.ts - t_ref <= max_future_sync_delta_s` 才选择队首数据，初始容差为 `0.05 s`；超过该容差则返回“不可同步”。

只有 `current_frame` 和 `odometry` 都成功选出样本时，才允许进入 ROI 筛选、平面估计和合成状态更新。任一队列返回“等待”时继续等待，不发布该基准帧；任一队列返回“不可同步”时，不使用当前帧生成新地面点，也不修改持久化合成地面状态，但必须使用当前真实 `global_map` 和已有历史合成地面构建并发布 `global_map_upsample_ground`，然后弹出该基准帧。该回退输出仍执行真实点优先的三维去重和障碍冲突检查，保证当前真实点不会被历史合成点覆盖。完成同步或确认不可同步后，应清理不再可能参与后续匹配的过旧样本，并弹出已处理的基准 `global_map`。

同步失败时的输出关系为：

`global_map_upsample_ground(t) = real_global_map(t) union validated_historical_synthetic_ground(t-1)`

其中不包含由当前帧新生成或更新的合成地面点。若当前同步数据不足以重新计算边界，可沿用最近一次有效边界裁剪历史合成点；无论边界是否可更新，都必须执行当前真实点冲突检查。

### 6.2 机器人相对感兴趣区域

- ROI 以同步后的机器人位姿为中心。
- 初始 XY 范围为 `5 m x 5 m`。
- 使用 `point.z - robot.z` 表示候选点高度，不使用绝对世界 Z。
- 根据录制的 M20 数据配置最小和最大相对高度。
- 删除 NaN、Inf、零距离点和明显的孤立离群点。
- 拟合前对候选点降采样，避免近距离传感器点密度主导拟合结果。

### 6.3 鲁棒 Z 直方图种子

- 使用 2% 到 5% 的低分位高度作为直方图起点，不使用绝对最低点。
- 使用 `z_bin_size_m` 对候选点按 Z 分箱，初始值为 `0.05 m`。
- 将点数最多的分箱作为候选种子，而不是最终地面分类结果。
- 将相邻的下一层和上一层加入拟合候选集合。
- 不能仅因为相邻分箱点数超过主分箱的 50% 就接受该分箱。
- 使用到平面的几何残差、XY 连通性和法向量一致性判断相邻点是否属于同一表面。

直方图步骤仅用于缩小 RANSAC 搜索空间，不能单独决定最终地面平面。

### 6.4 平面拟合

- 使用 RANSAC 拟合形式为 `z = a*x + b*y + c` 的平面。
- 使用鲁棒或加权最小二乘细化已接受的 RANSAC 内点。
- 计算内点数、内点比例、RMSE 和 XY 覆盖率。
- 拒绝近似线状的点集和集中在过少 XY 单元中的点集。
- 拒绝未包含机器人附近支撑区域或与该区域不连通的拟合结果。

只有所有质量门槛均通过时才接受一次拟合。拟合被拒绝的帧不生成新合成地面，也不修改已有状态。

### 6.5 Global Map 已探明边界

这里的“已探明边界”是禁止上采样点越过的外侧边界，不是要求 XY 栅格必须已经有真实点云覆盖。边界以内的未知区域仍允许生成上采样地面；这样可以补全尚未被点云直接击中的地面，同时避免在墙体后方等不可见区域凭空生成地面。每次融合前必须使用与 `global_map` 时间同步的机器人位姿执行以下步骤：

1. 对 `global_map` 中每个点计算相对高度 `relative_z = point.z - robot.z`。
2. 只保留 `relative_z_min_m <= relative_z <= boundary_relative_z_max_m` 的点，其中 `boundary_relative_z_max_m` 固定初始值为 `+0.50 m`。
3. 将保留点投影到 XY 平面，并按 `ground_resolution_m` 栅格化，提取由墙体、障碍物或观测范围终止形成的外侧边界。
4. 只闭合尺寸不超过 `boundary_max_gap_fill_m` 的小间隙，不能使用无约束凸包把边界跨到墙后、走廊外部或大面积不可见区域。
5. 分别保留投影形成的有效连通区域，并从栅格掩码提取外边界；内部未知空洞不是禁止区域。
6. 上采样候选单元只要位于允许的边界内部或边界上即可，不要求该 XY 栅格已有真实 `global_map` 点；越过外侧边界的候选单元直接拒绝。

最终允许生成合成地面的区域是三者交集：当前帧平面空间支撑区域、global map 外侧边界以内的区域，以及障碍物冲突过滤后的安全区域。边界以内的未知区域可以通过平面支撑掩码进行补全。

边界检查必须应用于全部持久化合成地面，而不只检查当前帧新增点。以前保存但已越过当前外侧边界的合成单元不得进入本次输出，可以暂时保留在内部状态中等待后续重新确认，也可以按过期策略清理。边界为空、点数不足或提取失败时，为避免无约束外推，本次输出不得包含任何合成地面，只能原样发布真实 `global_map`。

### 6.6 空间支撑掩码

模块不能无条件填满整个 `5 m x 5 m` 矩形。

- 将已接受平面的内点栅格化到世界 XY 地面网格中。
- 选择与局部地面支撑相关联的最大连通区域。
- 只填充直径小于 `max_hole_fill_m` 的内部小空洞。
- 向外膨胀距离最多为 `max_support_dilation_m`。
- 对不在当前平面支撑传播范围内的较大空洞和断裂保持无支撑；仅因 `global_map` 没有直接命中而形成的边界内未知区域，不得单独作为拒绝理由。
- 支撑传播遇到高度突变或已知障碍物列时必须停止。
- 即使无限平面经过某处，位于真实传感器支撑范围以外的单元也必须拒绝。

### 6.7 均匀地面生成

- 每个受支撑的世界 XY 单元生成一个候选点。
- XY 坐标放置在固定世界栅格上，保证重复经过时产生相同键。
- 在栅格坐标处根据已接受平面方程计算 Z。
- 状态键和输出间距均使用 `ground_resolution`，初始值为 `0.05 m`。
- 在条件允许时，为合成点保留独立的诊断表示。

完整支撑的 `5 m x 5 m` 区域在 `0.05 m` 间距下大约产生 10,000 个候选单元，正常情况下空间支撑掩码应进一步减少该数量。

## 7. 持久化地面更新规则

对每个生成的候选单元执行以下规则：

1. 当 `(ix, iy)` 尚不存在时插入该单元。
2. 当新旧高度差不超过 `max_ground_update_delta_m` 时融合高度。
3. 一致更新后增加置信度和观测次数。
4. 当高度差大于普通融合阈值时，仅在新拟合置信度明显更高的情况下替换旧单元。
5. 融合和替换条件均不满足时拒绝该不一致更新。
6. 只有接受更新后才刷新 `last_seen_ts`。

高度融合采用置信度加权。观测次数和置信度必须有饱和上限，保证旧单元在后续出现高质量观测时仍可被修正。

可选的置信度衰减机制可以在 `ground_stale_after_s` 后删除旧合成单元。设置为零表示在静态环境中禁用按时间过期。即使禁用过期，明确与旧地面矛盾的新观测仍必须能够删除或替换旧合成地面。

## 8. 真实地图与合成地图融合

每次输出都由最新真实快照和当前合成地面状态重新构建：

`output(t) = real_global_map(t) union synthetic_ground_state(t)`

严禁使用以下递归形式：

`output(t) = output(t-1) + global_map(t) + current_ground_patch(t)`

### 8.1 数据权威规则

- 真实射线地图点的权威级别高于合成地面。
- 同一 XY 单元中已存在接近合成高度的真实地面点时，省略合成点。
- 真实点表明机器人碰撞高度范围内存在低矮障碍物时，省略该 XY 单元的合成点。
- 真实点表明存在垂直墙体或地形突变时，在冲突区域及其配置的安全边距内省略合成地面。
- 只有高处点与地面的间隙超过 M20 共用的顶部净空阈值时，高处物体才能与地面共存。
- 绝不能为了合成地面而从 global map 快照中删除真实点。

这些规则是必要的，因为 `height_cost_occupancy` 使用每个 XY 单元中的最低和最高高度。如果盲目地在墙体或人下方插入地面，垂直高度差可能超过 `can_pass_under`，从而错误地将障碍物判定为可从下方通过。

### 8.2 最终去重

- 合成状态使用二维键 `(ix, iy)` 去重。
- 融合输出使用三维体素键 `(ix, iy, iz)` 去重。
- 真实点和合成点进入同一个三维键时保留真实点。
- 构建真实三维键哈希集合，仅追加集合中不存在的合成键。
- 不得反复使用 `PointCloud2.__add__` 作为累加器。

上游射线地图输出已经位于体素中心，因此使用相同的固定世界栅格可以最大程度减少近似重复点。

### 8.3 非覆盖式融合实现

上采样模块只能向射线地图结果补充点，不能覆盖、移动、平均或删除 `RayTracingVoxelMap.global_map` 中的任何真实点。具体实现必须遵守以下顺序：

1. 将当前 `global_map` 视为只读输入，不能原地修改其 NumPy 数组、Open3D tensor 或 `PointCloud2` 对象。
2. 以完整真实点集初始化本次输出，真实点的坐标、数量和三维体素键必须保持不变。
3. 根据完整真实点集建立三维体素键集合和 XY 障碍物冲突掩码。
4. 逐个检查合成地面点；与真实键重复或落入冲突区域的合成点直接丢弃。
5. 仅对上采样生成的地面点执行当前 `global_map` 外侧边界裁剪：删除越过边界的当前帧点和历史点；真实射线地图点不参与裁剪且必须原样保留。边界以内没有真实点云覆盖的区域不因“未知”状态被直接丢弃。
6. 只将通过边界和冲突检查的合成点追加到真实点集末尾，并据此创建新的 `PointCloud2`。
7. 不得对真实与合成的组合点集执行 column carving、体素质心平均、随机降采样或其他可能改变真实点的处理。
8. 持久化地面状态只保存合成地面单元，不保存也不更新真实射线地图点。

融合实现必须保持集合不变量：每个真实三维体素键都存在于输出中；新增的输出键只能来自通过冲突检查的合成地面。即使合成处理失败，也必须能够原样发布当前真实 `global_map`。

## 9. CostMapper 集成

`CostMapper` 当前通过 `global_map` 流名称匹配来消费 `RayTracingVoxelMap.global_map`。蓝图必须显式地将 `CostMapper.global_map` 重映射到 `global_map_upsample_ground`，避免直接输出和增强输出两个生产者之间产生歧义。

原始 `RayTracingVoxelMap.global_map` 继续保留，用于：

- Rerun 可视化；
- 上采样前后的结果对比；
- 地图质量指标统计；
- 在不改变 RayTracing 行为的情况下禁用该功能。

首个实现应受 M20 蓝图配置开关控制，并默认关闭，使原有流水线始终可以作为回退路径。

## 10. 建议配置

| 字段 | 初始值 | 用途 |
|---|---:|---|
| `enabled` | `false` | 上线控制开关 |
| `roi_size_x_m` | `5.0` | 局部拟合宽度 |
| `roi_size_y_m` | `5.0` | 局部拟合长度 |
| `relative_z_min_m` | 根据数据标定 | 相对里程计的最小候选高度 |
| `relative_z_max_m` | 根据数据标定 | 相对里程计的最大候选高度 |
| `boundary_relative_z_max_m` | `0.50` | 提取 global map 已探明边界时的相对高度上限 |
| `boundary_max_gap_fill_m` | `0.10` | 边界栅格中允许闭合的最大间隙 |
| `candidate_voxel_size_m` | `0.05` | 拟合前的密度归一化分辨率 |
| `z_bin_size_m` | `0.05` | 高度直方图分层间距 |
| `ground_resolution_m` | `0.05` | 持久化 XY 键和输出间距 |
| `plane_residual_threshold_m` | `0.04` | RANSAC 内点残差阈值 |
| `min_plane_inliers` | `200` | 最小有效内点数 |
| `min_plane_inlier_ratio` | `0.50` | 最小有效内点比例 |
| `max_plane_rmse_m` | `0.04` | 细化拟合的最大误差 |
| `min_xy_coverage_ratio` | `0.30` | 拒绝过度集中或线状的拟合 |
| `max_hole_fill_m` | `0.10` | 允许填充的最大内部空洞 |
| `max_support_dilation_m` | `0.15` | 允许超出支撑区的最大外推距离 |
| `max_ground_update_delta_m` | `0.08` | 已有单元的高度融合阈值 |
| `replacement_confidence_ratio` | `1.5` | 替换旧单元所需的置信度优势 |
| `obstacle_min_height_m` | `0.08` | 高于拟合地面的最小障碍物高度 |
| `max_future_sync_delta_s` | `0.05` | 辅助队列队首晚于 global map 时允许的最大时间差 |
| `current_frame_queue_size` | `20` | 当前帧点云时间戳有序队列的容量上限 |
| `global_map_queue_size` | `20` | global map 时间戳有序队列的容量上限 |
| `odometry_queue_size` | `200` | 里程计时间戳有序队列的容量上限 |
| `ground_stale_after_s` | `0.0` | 可选合成地面过期时间，零表示禁用 |
| `max_ground_cells` | `1_000_000` | 内存安全上限 |
| `publish_ground_patch` | `true` | 诊断输出开关 |

硬件上线前必须根据录制的 M20 点云和里程计高度分布标定相对 Z 范围，不能直接复制其他机器人参数。

## 11. 失败与回退行为

| 失败情况 | 必须执行的行为 |
|---|---|
| 里程计数据源不可用或已停止 | 跳过当前地面更新，输出当前真实 global map 与已有历史合成地面的融合结果；暂时尚未入队按同步规则等待 |
| 辅助队列最新时间仍早于基准 | 保留基准 global map，等待后续样本，不执行地面估计 |
| 辅助队列队首晚于基准超过 `0.05 s` | 不更新合成状态，立即输出当前真实 global map 与已有历史合成地面的融合结果，并消费该基准帧 |
| 候选点过少 | 跳过地面更新，保留已有合成状态 |
| 平面质量不合格 | 跳过地面更新，保留已有合成状态 |
| 外侧边界为空或提取失败 | 本次不发布任何合成点，原样发布真实 global map |
| 状态达到容量上限 | 停止插入新单元并按节流频率记录错误 |
| 坐标系不匹配 | 拒绝融合并按节流频率记录错误 |
| 内部异常 | 在条件允许时原样发布真实 global map |

模块必须以原始真实地图作为开放式回退。地面拟合失败绝不能阻塞代价地图发布。

## 12. 并发与背压

- 在三个独立的有界时间戳队列中保存 current frame、global map 和 odometry。
- 以 global map 队首为同步和输出基准，队列更新后尝试推进同步状态机。
- 使用锁保护地面状态更新和快照读取。
- 在锁外执行平面拟合，接受拟合后在一个短临界区内应用单元更新。
- 不得用 latest-wins 静默覆盖正在等待同步的 global map。队列达到容量上限且必须移除最旧基准帧时，应先将该帧与已有历史合成地面融合并输出 `global_map_upsample_ground`，但不执行当前帧地面补全或状态更新。
- 防止两个拟合结果以错误的时间顺序更新同一地面状态。
- 不在内存中保存完整历史点云。

## 13. 可观测性

模块必须发布或记录以下指标：

- 当前帧点数。
- ROI 和高度筛选后的候选点数。
- 主 Z 分箱和相邻分箱点数。
- 平面内点数和内点比例。
- 平面 RMSE。
- 受支撑的 XY 单元数。
- global map 边界筛选点数、边界面积和连通区域数。
- 因超出 global map 已探明边界而被抑制的当前与历史合成单元数。
- 插入、融合、替换、拒绝、过期和因冲突被抑制的地面单元数。
- 持久化地面单元总数。
- 真实点、合成点、去重点和最终输出点数。
- 当前帧、global map 和里程计之间的时间戳间隔。
- 三个同步队列的长度，以及同步成功、等待和不可同步的累计次数。
- 候选筛选、拟合、状态更新、融合、序列化和总处理延迟。
- 按拒绝原因分类的计数器。

Rerun 应使用独立颜色显示 `ground_patch`，并允许独立切换真实 global map 和增强地图的可见性。

## 14. 性能预算

在预期 M20 点云频率下的初始性能预算如下：

| 指标 | 目标 |
|---|---:|
| 地面估计 p95 | 不超过 `50 ms` |
| 状态更新 p95 | 不超过 `10 ms` |
| 真实/合成融合 p95 | 不超过 `50 ms` |
| 模块端到端 p95 | 不超过 `100 ms` |
| 完整支撑的 5 m 正方形内合成单元数 | 最大约 `10,000` |
| 持久化内存 | 受 `max_ground_cells` 限制 |
| 重复经过后的输出增长 | 收敛，不随经过次数线性增长 |

如果重建完整增强点云成为主要耗时，应先优化键生成和集合成员判断，再考虑增加线程。完整地图融合可以使用 O(N) 哈希并集时，应避免使用基于排序的 `numpy.unique`。

## 15. 测试计划

### 15.1 单元测试

- 正坐标和负坐标下的世界栅格键生成。
- 同一 XY 重复插入后只保留一个单元。
- 一致的重复观测融合高度且不增加状态大小。
- 不一致的重复观测按置信度规则拒绝或替换。
- 观测次数和置信度饱和。
- 真实三维体素覆盖合成重复体素。
- 高度范围外的 global map 点不参与已探明边界生成。
- 边界外的当前和历史合成点均不会进入融合输出。
- 凹形边界和内部大空洞不会被无约束凸包填平。
- 边界为空或无效时，输出只包含真实 global map。
- 融合函数不修改输入 `global_map` 对象或其底层点数组。
- 融合前后的真实点数量、坐标和三维体素键完全一致。
- 合成点与真实点冲突时只删除合成点，不删除或移动真实点。
- 低矮障碍物冲突抑制合成地面。
- 辅助队列跨越基准时间时，从前后两个样本中选择时间差较小者。
- 辅助队列末尾仍早于基准时间时保持等待且不消费基准 global map。
- 辅助队列队首比基准晚 `0.05 s` 以内时允许同步；超过 `0.05 s` 时不更新状态，但输出仍包含当前真实 global map 和已有历史合成地面。
- 三个队列收到乱序数据后仍按消息时间戳完成匹配。
- 容量限制阻止无界插入。
- 空点云和非法点云回退到真实地图。

### 15.2 地面估计器测试

- 带高斯噪声的平坦地面。
- 带孤立低位离群点的平坦地面。
- 不具备足够 XY 覆盖率的稀疏扫描线。
- 平坦地面上的低矮障碍物。
- 穿过拟合区域的行人。
- 与地面区域相交的墙体。
- 必须保持无支撑状态的坑洞或落差边缘。
- 完全不可见的地面。
- 两个分离平面区域中只接受与机器人连通的区域。

### 15.3 集成测试

- 使用内存传输构建 `UpsampleGround` 模块。
- 输入同步的当前帧、global map 和里程计消息。
- 验证输出时间戳和坐标系传播。
- 验证融合地图原样保留真实点。
- 验证合成地面只填充有支撑的小空洞。
- 验证所有合成点都位于 global map 外侧边界内部或边界上；边界以内的未知栅格允许存在合成点。

- 验证禁用功能后恢复原有直接 global map 路径。

### 15.4 回放验证

- 通过坡道和门槛。

## 16. 验收标准

- 三个输入队列始终按消息时间戳升序匹配，以 global map 队首时间戳作为基准。
- 辅助队列跨越基准时间时选择前后两个样本中时间差更小的样本。
- 辅助队列尚未出现晚于基准的样本时保留基准帧等待，不执行地面估计。
- 辅助队列队首晚于基准超过 `0.05 s` 时，不补全当前帧且合成地面状态保持不变，但 `global_map_upsample_ground` 仍包含已有历史合成地面。
- 每个世界 XY 单元最多包含一个合成地面点。
- 重复经过相同路线不会导致合成点数线性增长。
- 状态收敛后，在没有新增真实观测范围的情况下，再次经过相同路线导致的合成单元总数变化不超过 5%。
- 真实 global map 点在 float32 容差内按字节或数值保持不变。
- 输出包含输入 global map 的全部真实三维体素键，不允许缺失。
- 上采样处理不得覆盖、平均、移动或删除任何 RayTracing 真实点。
- 每个输出合成点都位于由 `[relative_z_min_m, +0.50 m]` 高度范围内 global map 点投影形成的已探明 XY 边界内。
- 当前边界以外的持久化历史合成点不会进入输出。
- 任何合成点都不会输入 `RayTracingVoxelMap`。
- 超出残差、内点数或覆盖率限制的平面拟合不会添加地面。
- 人工标注的障碍物和落差边缘不会被转换为可通行填充区域。
- 在回放验证集中，增强代价地图提高已知地面覆盖率，同时不降低障碍物召回率。
- 模块 p95 处理时间满足性能预算。
- 禁用模块后，无需修改其他代码即可恢复当前 M20 导航行为。

## 19. 核心设计决策

- 单层地面允许使用唯一 `(ix, iy)` 状态键。
- 真实 global map 在 `UpsampleGround` 中具有权威性且不保存内部历史。
- 只有合成地面跨帧持久化。
- 每次根据当前输入重新构建融合输出，绝不递归累加。
- 平面拟合必须通过鲁棒几何校验。
- 地面生成范围受真实观测的空间支撑限制。
- 合成地面由 `CostMapper` 消费，不进入 `RayTracingVoxelMap`。
- 同步不确定时不创建当前帧的新地面，但继续用已确认的历史合成地面构建 `global_map_upsample_ground`。
