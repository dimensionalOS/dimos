Strategy

Judges are the Dimensional team. The deliverable is a 90-second video. The event is a 3-day interview. So: optimize for a tight, robust, narratable behavior — not maximal autonomy. The work lives in Skills (RPC functions the LLM agent calls), not in rewriting the nav stack. The demo must read in 90 seconds and degrade gracefully when hardware misbehaves.

💰 Alipay payments via MCP

DimOS agents speak MCP; Alipay ships an official MCP server. Wire both into one agent — one MCP drives the Go2, the other takes real payments. The robot can reason, move, and transact.

# official global repo (Python 3.11+)
uvx ant-intl-alipayplus-mcp
# tools: create_payment · query_payment · cancel_payment · create_refund · customs_*
# env:   CLIENT_ID · MERCHANT_PRIVATE_KEY · ALIPAY_PUBLIC_KEY · PAYMENT_NOTIFY_URL

Demo loop: create_payment → robot shows QR / AI Pay → query_payment polls until paid → robot completes the physical action.

⚠ One real blocker: credentials. Need merchant CLIENT_ID + RSA keys, no obvious public sandbox. Ask Ant/Dimensional mentors for sandbox creds at check-in. Keep a mock payment tool as fallback. Note: a mainland-China Alipay AI-Pay MCP also exists — likely better for Shanghai.

Gotchas (hour-eaters)

  • Transforms/TF are the #1 bug — wrong frame = wrong direction. Sanity-check in Rerun.
  • ROS2 + Docker setup is non-trivial — do it before the event, not Day 1.
  • VLM/LLM latency — describe at stops only, tiny prompts, cache. Never per-frame.
  • Local-network only — robot link is WebRTC/ROS2 on the venue LAN.
  • LLM key + credits — the agent is dead without an authenticated, funded key.

Pitches (100 words)

Our agent turns the Go2 into an autonomous security patrol that thinks out loud. It walks a set of waypoints; at each stop a VLM describes the scene and compares it to a learned "normal" in spatial memory. When something's off — a door left open, an unfamiliar person — the agent reasons about it, narrates it aloud, captures the frame, and logs an incident with location and timestamp. If it gets stuck, the LLM diagnoses the failure and replans. Built entirely as DimOS Skills.
We turn a pack of Go2s into a single choreographed performer. One DimOS Blueprint deploys the same behavior file across every dog, and a conductor agent issues natural-language cues — "form a line," "wave down the row," "scatter and regroup" — which each robot interprets relative to its own pose and neighbors. Because the logic is one file fanned out by autoconnect, adding a sixth dog is a config change, not a rewrite. A live demo that DimOS's multi-robot abstractions actually scale.
