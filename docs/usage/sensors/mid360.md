# How to Use and Setup Mid-360
The Mid-360 uses an M12 Aviation Connector for both power and data. If you do not have the Livox 1-to-3 spliter cable please refer to their official documentation to find the pinout information. To run the Mid-360 blueprints the Mid-360 needs to be hooked up to power (supports 9V to 27V) and to an ethernet port on your host machine.

## Setting up the Ethernet Connection
Whether you are connecting the Mid-360 directly to your computer's ethernet port or using a USB to Ethernet adapter, you need to assign a static IP address to that interface so it can communicate with the LiDAR.

- Run `ip -br link` to find the name of your network interface.
  - For a direct ethernet port, it will typically look like `eth0`, `eno1`, or `enp4s0`.
  - For a USB to Ethernet adapter, it will typically look like a long MAC-address-based name such as `enx00e04c680125`.
- Assign a static IP to the interface using:
  `sudo ip addr add 192.168.1.5/24 dev <interface_name>`

- Make sure the ethernet cable is fully connected and verify the connection by pinging the default Mid-360 IP address:
  `ping 192.168.1.155`

Now you should be able to use any Mid-360 blueprint.

### Finding the LiDAR IP

The Mid-360's default IP address is `192.168.1.155`, which is what the `mid360-fastlio` blueprints use by default. You can confirm the sensor is reachable at that address by pinging it:
  `ping 192.168.1.155`

If your sensor has a different IP, you can find it by scanning the subnet after setting up the ethernet connection. Either of these commands will list devices on the `192.168.1.x` subnet:

```bash
# Option 1 — arp-scan (install with: sudo apt install arp-scan)
sudo arp-scan --interface=<interface_name> --localnet

# Option 2 — nmap (install with: sudo apt install nmap)
nmap -sn 192.168.1.0/24
```

The Mid-360 will appear as a Livox device in the output. Once you have the IP, you will need to pass `lidar_ip` when constructing the `FastLio2` blueprint. For example, in `fastlio_blueprints.py`:

```python
mid360_fastlio = autoconnect(
    FastLio2.blueprint(voxel_size=voxel_size, map_voxel_size=voxel_size, map_freq=-1, lidar_ip="192.168.1.200"),
    ...
)
```

## Livox Mid-360 dimos run
All of these will display data with Rerun

- `dimos run mid360` — raw LiDAR
- `dimos run mid360-fastlio` — SLAM with 0.15m voxel filtering
- `dimos run mid360-fastlio-voxels` — SLAM + downstream VoxelGridMapper
- `dimos run mid360-fastlio-voxels-native` — SLAM + C++ global voxel map at 3Hz
